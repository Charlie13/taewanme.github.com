1.3 OCI CLI 설정 기본 정보 수집

oci cli는 Oracle Cloud Infrastructure에서 주요 인프라 관리 기능을 수행하는 CLI(Command Line Interface) 툴입니다. “OCI CLI“를 사용하여 OCI 웹 콘솔과 같은 기능을 수행할 수 있습니다. “OCI CLI“를 스크립트와 함께 사용할 경우 OCI 인프라 관리 기능을 자동화하고 편리하게 운영할 수 있습니다. Oracle Cloud Infrastructure를 더 효율적으로 사용하기 위해서 Command Line Interface(이하 CLI) 사용이 필수입니다.

“OCI CLI“를 설치하고 Oracle Cloud 계정과 연결하기 위해서는 다음과 같은 정보가 필요합니다.

  • OCI User의 OCID
  • OCI Tenancy ID
  • OCI Region

이 정보는 확인하는 방법을 정리합니다.

OCI User의 OCID

OCI User의 OCID는 OCI User 관리 페이지에서 확이할 수 있습니다. OCI Compute 관리 페이지에서 다음 메뉴로 “OCI User” 관리 페이지로 이동합니다.

  • “MENU” -> “Identity” -> “Users“

다음 그림과 같이 OCI User 관리 페이지에서 OCI CLI 접속에 사용할 OCI User의 OCID를 확인할 수 있습니다. OCID 문자열에 “copy” 링크를 클릭하면 OCID 문자열이 복사됩니다.

  • OCI User(adw_hol_user)의 ocid

OCI Tenancy ID

OCI Tenancy의 OCID는 OCI Tenancy 관리 페이지에서 확이할 수 있습니다. OCI Compute 관리 페이지에서 다음 이미지와 같이 오른쪽 위의 사용자 아이콘을 클릭한 후 나타나는 “Tenancy” 메뉴를 클릭하면 “OCI Tenancy” 관리 페이지로 이동합니다.

  • OCI Tenancy 관리 페이지 이동 메뉴

다음 이미지와 같이 “OCI Tenancy” 관리 페이지에서 OCI Tenency의 ocid와 Region 정보를 확인할 수 있습니다.

  • OCI Tenancy ocid와 Region
